2. `git rev-parse HEAD` recorded as the tag target.
3. `PYTHONPATH=src python3 -m pytest -q` → **300 passed** (or higher if tests were

7. No uncommitted changes; no untracked source files intended for the release.

**If any check fails: stop, do not tag, report.**

## Post-tag verification

1. `git tag -n v0.2.0-p9-persistent-history` shows the annotated message.
2. `git rev-parse v0.2.0-p9-persistent-history^{commit}` == the recorded merge
   commit.
3. After `git push origin v0.2.0-p9-persistent-history`: the tag is visible on the
   remote and points at the same commit.
4. Re-run checks 3–5 above **from a fresh checkout of the tag** to confirm the
   release point reproduces the golden world and transcript.
5. Record in the release notes / memory that **seed42 replay is now the frozen

## Rollback note

Tags are cheap and movable before push. If a post-tag check fails, delete the
local tag (`git tag -d …`) and re-cut after fixing — do **not** push a tag that
fails verification.
